# PRD: Web-приложение для спортивного ориентирования (MVP v1)

## 1. Vision и цель
Создать web-приложение, которое помогает быстро строить и сравнивать маршруты по загруженной карте местности.
Ключевой сценарий MVP: `загрузка карты -> выбор точек -> расчет и сравнение маршрутов`.

## 2. Целевая аудитория
- Спортсмены по ориентированию
- Тренеры и клубы
- Организаторы тренировок/соревнований

Примечание: в MVP функциональность для этих групп одинаковая.

## 3. Основная ценность
- Автоматизация анализа карты и построения маршрутов
- Сокращение времени выбора маршрута
- Прозрачность выбора (почему маршрут лучше)
- Накопление истории решений по версиям карты

## 4. Scope MVP (in)
- Регистрация/авторизация
- Личный кабинет пользователя
- Загрузка карты (`PNG/JPEG`)
- Автоматическая оцифровка объектов карты
- Ручное редактирование результатов оцифровки
- Построение графа проходимости
- Поиск маршрутов между заданными точками
- Вывод `top-3` маршрутов с объяснением различий
- Исключение похожих маршрутов (обеспечение разнообразия)
- История запусков маршрутизации с привязкой к версии карты
- Экспорт маршрута в `PNG/PDF`
- CRUD типов пользовательских объектов местности

## 5. Out of scope (после MVP)
- Геопривязка
- Мобильное нативное приложение
- Совместная real-time работа над одной картой
- Онлайн-дообучение ML на пользовательских правках
- Интеграции с GPS-часами

## 6. Функциональные требования
### 6.1 Аккаунт и доступ
- Регистрация, логин, восстановление пароля
- Роли: `User`, `Admin`
- Строгая изоляция данных по аккаунтам

### 6.2 Карты
- Загрузка `PNG/JPEG`
- Хранение оригинала и рабочей версии
- Статусы карты: `Uploaded`, `Digitized`, `Edited`, `Ready`
- Версионирование: изменения оцифровки создают новую версию

### 6.3 Оцифровка карты
- Подход MVP: гибрид `классическое CV + ручная правка`
- 5 обязательных классов объектов:
  - Растительность
  - Водные
  - Скальные
  - Земельные
  - Искусственные
- Для каждого класса поддержать 3 геометрии: `точка`, `линия`, `область`
- Для каждого объекта хранить: класс, тип, геометрию, коэффициент проходимости, источник (`auto`/`manual`)
- История правок и версий обязательна

### 6.4 Редактор оцифровки
- Операции: добавить/удалить/изменить объект
- Редактирование геометрии (точка/линия/полигон)
- Изменение коэффициента проходимости
- Undo/redo

### 6.5 Маршрутизация
- Взвешенный граф из оцифрованной карты
- Поиск маршрутов по заданным точкам
- Приоритет оптимизации: `время прохождения + безопасность`
- Поддержка профиля спортсмена (MVP: базовый профиль, расширение позже)
- Расчет `top-3` маршрутов
- Разнообразие маршрутов обязательно (похожие исключаются)

### 6.6 Объяснимость маршрута
Для каждого маршрута показывать:
- Оценку времени прохождения
- Интегральный риск/безопасность
- Суммарный штраф
- Краткое объяснение выбора

### 6.7 История и повтор
- Сохранение каждого запуска маршрутизации
- Привязка к: карте, версии карты, настройкам, времени
- Возможность повторного запуска с историческими параметрами

### 6.8 Экспорт
- Экспорт в `PNG` и `PDF`
- В экспорт включать: карту, маршрут, точки, краткую легенду

## 7. Нефункциональные требования
### 7.1 Производительность
- Поиск маршрута: `< 10 сек` для типовой карты
- Расчет `top-3`: `< 10 сек` для типовой карты
- Во время расчета обязателен `progress bar`

### 7.2 Качество оцифровки
- Целевая точность: `80%+`
- Метрика качества MVP:
  - Macro F1 по 5 классам >= 0.80 на валидационном наборе
  - Геометрическая точность (IoU) для областей >= 0.70

### 7.3 Надежность
- API uptime: `99.5%+`
- Ежедневные бэкапы БД

### 7.4 Безопасность
- JWT + refresh tokens
- Хэширование паролей (`Argon2` или `BCrypt`)
- Валидация загрузок по mime/размеру

## 8. UI/UX требования (desktop-first)
- Desktop-first интерфейс (мобильная версия позже)
- Основной экран:
  - Центральная область: карта
  - Правая панель: список выбранных точек, параметры и описание маршрута
- Слои карты: исходник / оцифровка / граф / маршрут
- Визуальная индикация стоимости и рисков на маршруте
- Тема: темная, минималистичная, красно-оранжевый акцент

## 9. Технический стек MVP
- Frontend: `React + TypeScript + Vite`
- Canvas/редактор: `react-konva` (без геопривязки)
- Backend: `.NET 8 (ASP.NET Core Web API)`
- БД: `PostgreSQL`
- Кэш/очереди: `Redis` (опционально)
- Файлы: S3-совместимое хранилище или локально для MVP
- Auth: JWT + refresh
- Export PDF: серверно (`QuestPDF` или эквивалент)

## 10. Архитектура (высокоуровнево)
- `Frontend (React)` -> `ASP.NET Core API`
- Основные сервисы:
  - Auth
  - Map
  - Digitization
  - Routing
  - History
  - Export
- Хранилища:
  - PostgreSQL (доменные данные)
  - Object Storage (изображения)
  - Redis (кэш/задачи)

## 11. Модель данных (минимум)
- `User`
- `Map`
- `MapVersion`
- `TerrainObject`
- `TerrainObjectType`
- `GraphNode`
- `GraphEdge`
- `RouteRequest`
- `RouteResult`
- `RouteSegment`
- `Checkpoint`

## 12. API (черновой набор)
- `POST /auth/register`
- `POST /auth/login`
- `POST /maps/upload`
- `POST /maps/{id}/digitize`
- `PUT /maps/{id}/objects`
- `POST /routes/calculate`
- `GET /maps/{id}/routes/history`
- `POST /routes/{id}/export?format=pdf|png`
- `POST /terrain-types`
- `GET /terrain-types`

## 13. Метрики успеха
- Time-to-route (медиана): `< 10 сек`
- Точность автооцифровки: `80%+`
- Доля построений с экспортом: `> 40%`
- Ошибки API: `< 1%`

## 14. Ключевые риски и меры
### Риск 1: неточная оцифровка
- Меры:
  - Гибридный подход (CV + ручная правка)
  - Эталонный валидирующий набор карт
  - Версионирование правок

### Риск 2: неверный маршрут
- Меры:
  - Проверяемая функция стоимости
  - Объяснимость итогового выбора
  - Интеграционные тесты на эталонных кейсах

### Риск 3: похожие top-3 маршруты
- Меры:
  - Порог похожести (например, overlap ребер)
  - Повторный поиск с штрафом на уже выбранные сегменты

## 15. Roadmap
1. Этап 1 (4-6 недель): Auth, загрузка карт, базовая оцифровка, маршрут между точками, история
2. Этап 2 (3-4 недели): редактор оцифровки, top-3 и объяснения, пользовательские типы объектов
3. Этап 3 (2-3 недели): экспорт PNG/PDF, улучшение UX, оптимизация производительности
4. Этап 4 (ongoing): повышение точности оцифровки, метрики и аналитика

## 16. Критерии приемки MVP
- Пользователь может зарегистрироваться и войти
- Пользователь загружает карту (`PNG/JPEG`) и получает автооцифровку
- Пользователь редактирует оцифровку и сохраняет версию
- Система строит 3 разнообразных маршрута и объясняет различия
- Время расчета маршрута не превышает 10 секунд на типовом кейсе
- Все расчеты сохраняются в историю с привязкой к версии карты
- Пользователь экспортирует результат в `PNG` и `PDF`
- Интерфейс соответствует desktop-first UX с центральной картой и правой рабочей панелью
